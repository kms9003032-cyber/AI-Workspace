## 2. 부품 호환성 불확실

부품이 도착하더라도 실제 조립 시:

- 치수 오차
- 체결 문제
- 간섭 문제

- 그리퍼 장착 문제

가 발생할 수 있다.

현재는 CAD 기준으로 설계했지만,  
실제 조립 후 맞는지는 직접 확인해야 한다.

---

## 3. 프로파일 맞춤가공 문제

현재 프로파일 맞춤가공이 새로운 문제로 발생하였다.

문제점:

- 구멍 가공 가격이 비쌈
- 원하는 날짜보다 늦게 도착할 가능성 있음
- 일정 지연 시 전체 조립 일정에 영향 발생

따라서 대안 검토가 필요하다.

가능한 대안:

- 직접 가공 가능 여부 확인
- 가공 위치 최소화
- 체결 방식 변경
- 임시 브라켓 사용
- 3D 프린팅 부품 활용
- 가공이 꼭 필요한 부품과 아닌 부품 분리

# Software Integration Issues

## 1. 코드 연동 우려

현재 각 기능의 코드를 서로 다른 사람이 작성하고 있다.

따라서 다음 문제가 발생할 수 있다.

- 입력 데이터 형식 불일치
- 출력 데이터 형식 불일치
- 좌표계 기준 불일치
- 파일명 및 변수명 불일치
- 실행 순서 불명확
- Arduino 명령 형식 불일치

가장 큰 우려는:

> 각자 만든 코드가 실제로 하나의 시스템처럼 연결될 수 있는가?

이다.

---

## 2. 필요한 해결 방향

코드 연동을 위해 반드시 정해야 할 것:

- Vision 출력 형식
- 체스 좌표 형식
- 물리 좌표 형식
- Arduino 명령 형식
- 모터 이동 명령 형식
- 오류 발생 시 처리 방식

예시:

```text
Vision Output:
from_square = "e2"
to_square = "e4"

Translation Output:
start_position = (x1, y1, z1)
end_position = (x2, y2, z2)

Arduino Command:
MOVE X Y Z
GRIP CLOSE
MOVE X Y Z
GRIP OPEN
```

# AI Issues

## 1. Validation Accuracy 부족

현재 AI 모델의 validation accuracy가 충분히 안정적이지 않다.

문제:

- val_accuracy가 낮음
- 학습 결과가 안정적으로 유지되지 않음
- 실제 사용 환경에서 성능이 더 낮아질 가능성 있음

---

## 2. Loss 문제

현재 loss 값이 크거나 불안정한 문제가 있다.

이는 모델이 단순히 정답률만 높은 것이 아니라,  
확신도 측면에서 아직 불안정할 수 있음을 의미한다.

---

## 3. unknown_or_empty 학습 부족

현재 unknown_or_empty 클래스 학습이 부족하다.

문제:

- 빈 공간을 기물로 오인할 가능성
- 알 수 없는 물체를 기존 체스말로 분류할 가능성
- confidence가 낮은 사진을 잘못 확정할 가능성

중요 주의사항:

> results 폴더에 자동 분류된 이미지를 그대로 학습에 사용하면 안 된다.  
> 반드시 사람이 확인한 뒤 올바른 train/val 폴더로 옮겨 재학습해야 한다.

---

## 4. 그리퍼 카메라 실사용 정확도 미지수

현재 AI가 학습 환경에서 어느 정도 성능을 보이더라도,  
실제 그리퍼 카메라에서 같은 정확도를 보일지는 아직 확실하지 않다.

우려점:

- 카메라 화질 차이
- 초점 거리 차이
- 조명 차이
- 그리퍼 그림자
- 체스말 각도 차이
- 배경 차이
- 흔들림

따라서 실제 그리퍼 카메라로 촬영한 데이터를 반드시 추가 테스트해야 한다.

# Integration Bottleneck

현재 가장 큰 병목은 전체 시스템 연결이다.

현재 연결해야 할 흐름:

```text
상부 카메라
↓
체스판 변화 감지
↓
체스 좌표 생성
↓
체스 엔진
↓
translation.py
↓
Arduino
↓
모터 제어
↓
그리퍼 동작
```

현재는 각 부분이 따로 존재하지만,  
하나의 시스템으로 완전히 연결되지는 않았다.
